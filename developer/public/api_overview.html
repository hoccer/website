Developer APIs

General

After installing a Linccer client on some device, the application code should generate a unique id which should be used throughout all sessions.

Continious Environment Updates (should happen at least every ten seconds):

PUT /v3/clients/[client_id]/environment
{  gps: {"latitude": 154.123, "longitude": 123456, 
           "accuracy": 300, timestamp: _unix timestamp_ },
   network: {"latitude": 154.123, "longitude": 123456, 
                 "accuracy": 300, timestamp: _unix timestamp_ },
   wifi: {bssids: [123:1231:hbsa, 123:23:dfb], 
           timestamp: _unix timestamp_ },
   latency: 300,
   client_name: "robert's iphone"
}
---
201 Created

Environment Update on nonexisting client id:

PUT /v3/clients/[non_existing_client_id]/environment
{  ... }
---
412 Precondition Failed (if id is no uuid)

Closing the Client (sign off)

DELETE /v3/clients/[client_id]/environment
---
200 OK

Peek

Peek allows you to get information about the client’s group (the clients nearby).
If you pass the group hash in the request, the request is held until
the group changed.

GET /v3/clients/[uuid]/peek?group_id=[group_hash]
---
200 OK
{
    group_id: "[group_hash]",
    group: [
       {name: "robert's iphone", id: "[client_id]"},
       {name: "john's iphone", id: "[client_id]"},
    ]
}

Sharing

Succsessfully sharing data:

PUT /v3/clients/[client_id]/action/[ one-to-one | one-to-many | many-to-many ]
{... json data ...}
---
200 OK
{... json data ...}

When no other clients has a similar envrionment or does not perform the counter part action in time

PUT /v3/clients/[client_id]/action/[ one-to-one | one-to-many | many-to-many ]
{... json data ...}
---
204 No Content

When environment is insufficient (none provided or to inaccurate)

POST /v3/clients/[client_id]/action/[ one-to-one | one-to-many | many-to-many ]
{... json data ...}
---
412 Precondition Failed

Multiple incompatible actions at the same time (eg. collision)

PUT /v3/clients/action/[ one-to-one | one-to-many | many-to-many ]
---
409 Conflict
