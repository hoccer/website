Hoccer API

The Linccer API

The Linccer API is intended to connect arbitrary devices on the same location. The devices do not need to be in the same local network or use a common connection type, they just need to share the same (geo) environment. Which part should you use?

Depending on your needs, you have to decide which parts to use.

You want to connect your Apps

The Linccer API allows to transfer arbitrary JSON data between the grouped devices. This offers high flexibility what to share. Just share the data that fits best to your app. If you write an app for passing urls you can just send
{ url : “http://hoccer.com” }.
But remember: to keep the sharing experiance smooth and fast, the JSON should not be to large. If you want to pass larger content like images you should look at our FileCache API.

You want to offer a file sharing app.

In addition to the plain linking API we offer a Hoccer spec. Hoccer is our populare file sharing app available for iPhone and Android. It aimes to simplify data sharing between devices. All Apps complying with the hoccer spec can exchange data between each other.

Actions

A Linccer transaction consists of two actions, a send action and a receive action. E.g. the Hoccer App uses the throw and catch methaphors for triggering these actions.
Currently both actions have to be performed (nearly) simultaniously. The way to trigger the actions is up to you. Some Apps may use buttons and others allow the users to bump their phones togeher.

Action Modes

Actions modes determine how many people can can share and receive simultaniously.

One-to-many

A one to many connection lets you distribute content from one device to several others.

One-to-one

This mode ensures a one-to-one transaction.

RESTful API

The Linccer API is a RESTful webservice. So if your favourite language has a http library (most have ;)) you can easily connect to the our service.
For convienence we offer higher level APIs in Objective-C, Java, Ruby or JavaScript.

Even when you plan to use one of the higher level api clients, you should read the RESTful Flow. It will give you a better understanding of how the linccer api works.

RESTful Flow

PUT http://linccer.hoccer.com/v3/clients/[client_id]/environment

As mentioned above hoccer groups clients based on the environment. Currently an environment consists of the clients location (latitude, longitude, accuracy) and the bssids around the client.
To allow a fast coupling groups are created before the exchange is triggered. You must PUT the environment to our server and you are only allowed to exchange data afterwards.
You should also continue to watch changes in the environment and send another PUT request for each change.
Our Hoccer App starts sending the environment just after the startup. For other apps it might be more appropriate to start the environment updates when the user enters a sharing mode, but that depands on the type of your application.

PUT http://api/v3/clients/[client_id]/action/[ one-to-one | one-to-many]

To send data PUT a valid JSON string to the action uri. Send the data as JSON in the Request-Body.

GET http://api/v3/clients/[client_id]/action/[ one-to-one | one-to-many]

To receive data perform a GET request on an action uri.
The arrival order on the server is not importend. They just hava to arrive within three seconds.

How to continue?

If you want to write a Linccer Client in another language read the complete RESTful specification.

If you want to use Linccer in your app, read the appropriate reference (iOs, Java, JaveScript).

If you want to connect your app with the Hoccer File Sharing Ecosystem, read the Hoccer spec.
